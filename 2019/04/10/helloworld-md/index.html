Kafka Introduction

NOTE: Version 1.0.1

Apache Kafka is a distributed streaming platform (分布式流处理平台).

What does distributed streaming platform mean?

A streaming platform has 3 key capabilities:

  • pub & sub to streams of records (message queue or enterprise messaging system).
  • store streams of records in a fault-tolerant way.
  • process streams of records as they occur.

Kafka’s 2 broad classes of application (两大类应用场景):

  • real-time streaming data pipelines (实时流数据通道), reliably get data between systems or applications (能够在系统或者应用之间进行可靠的数据传输）
  • real-time streaming application (实时流处理应用程序), transform or react to the streams of data (对流数据进行转换或者响应式操作)

Kafka in Detail

Some concepts first:

  • run as a cluster on 1 or more servers that can span multiple datacenters.
  • store streams of records in categories called topics.
  • each record contains of a key, a value & a timestamp.

4 core APIs:

  • Producer API: publish a stream of records to one or more Kafka topics
  • Consumer API: subscribe to one or more topics and process the stream of records
  • Streams API: allows an application to act as a stream processor, consuming an input stream from one or more topics and producing an output stream to one or more output topics, effectively transforming the input streams to output streams
  • Connector API: allows building and running reusable producers or consumers that connect Kafka topics to existing applications or data systems
