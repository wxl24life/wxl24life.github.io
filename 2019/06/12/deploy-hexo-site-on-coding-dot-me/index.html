将博客同步部署到 github 和 coding 平台

发表于 2019-06-12 | 阅读次数:

最近发现 hexo + github pages 搭建完博客后，通过国内网络访问很慢。同时，还存在另一个普遍反映的问题：没有办法被百度收录。据说是因为百度嫌 github pages 的打开速度太慢（还是网络的锅）。由于自己写的文章主要使用中文，如果没办法通过百度搜索到，会大幅降低自己希望通过写博客传播技术价值的初衷（一个小小的梦想 (⊙﹏⊙) b）.

诚然，写博客的主要目的是为了记录日常学习工作生活中的一些想法以及解决问题的思路，但是如果文章能够帮助到别人，肯定也是一件很有成就感的事情。尤其，当意识到会有更多的人读到自己写的文章，本身对于文章的写作质量自然会有更高的要求。

针对如何提升博客打开速度的问题，网上搜到的经验普遍是将整个博客站点往国内的代码平台同步托管一份，例如 coding.me。

这两天，我按照其他人总结的经验逐步做了设置，但是，经过各种尝试都没能通过自己的独立域名打开博客。最后，我对域名解析以及 CNAME 等网络知识做了简单 调研，成功将域名解析到了 coding pages 页面。

在此，我对整个过程里涉及到的操作步骤做一下记录。

首先，注册 coding.me，创建项目，添加公钥，修改 _config.yml 支持同步往 github.io 和 coding.me 上面推送代码，开启 coding pages。网上有更详细的操作步骤 介绍。

接下来是域名解析部分，几个关键步骤：

  1. hexo 站点目录 source 文件夹下添加 CNAME 文件，里面输入自己的域名。我填写的是 wxl24life.com，也可以写 www.wxl24life.com, 这两个是有区别的。

  2. 域名服务提供商网站上面添加 CNAME 记录。我用的是腾讯云，具体位置：云解析 -> 域名解析列表 -> 添加记录。

    • 我查阅到的所有文章都将主机记录即域名前缀设置成了 www。这里有坑， 如果设置成 www，后面使用域名时，必须带 www 前缀，例如 www.wxl24life.com，如果域名链接里不包含 www 将无法打开博客。
    • 如果希望通过不带 www 前缀的域名访问博客站点，需要在域名解析列表里将主机记录设定为 @。
    • 如果想支持 www 和不带 www 前缀两种域名访问方式，需要添加两条记录。且将前缀分别设定为 www 和 @。
  3. coding.me 平台 Pages 服务绑定在自定义域名，为了能够支持通过 www 和非 www 两种前缀访问博客站点，绑定了两个域名，分别是 www.wxl24life.com 和 wxl24life.com。

  4. 通过 dig 工具可以验证 CNAME 是否设置生效。

把遇到的坑总结如下：

  1. 域名解析添加主机记录时，参考其他文章，使用了 www 前缀
  2. coding pages 服务里面绑定域名时，绑定了不带 www 前缀的域名，即 wxl24life.com
  3. dig 验证时，一直在验证不带前缀的 wxl24life.com，一直不成功

回过头来看，其实腾讯云域名解析记录添加时，有对 @ 和 www 两种前缀的详细解释：

提示

要解析 www.wxl24life.com，请填写 www。主机记录就是域名前缀，常见用法有：

www            解析后的域名为 www.wxl24life.com
@            直接解析主域名 wxl24life.com

如果对这里的提示认真理解一下，而不是盲从其他人的操作设置，可能会少走很多弯路。通过这个案例也再次提醒我们，网上的经验文章不一定完全适用于自己，如果只是盲从完全照搬的话，踩坑的可能性会很大。

当然，最后这部分的反思也适用于我博客里所谓的 “经验” 文章，仅做参考。
