GitHub 上 Fork 和 Star 两个操作的解释

发表于 2019-07-12 | 阅读次数:

先抛出两个问题：

  1. star 和 fork 到底啥区别
  2. fork 来的项目，在其之上做 commit，默认不会统计到个人页面的贡献值里

问题 1

GitHub 个人 profile 页面里有 Repositories 和 Stars 两个列表。

  • Repositories 里既有自己创建的 repo，也包括了 fork 来的 repo
  • Stars 列表里包括了所有 star 过的 repo

就我而言，一直以来都有一个疑惑：为什幺把 fork 来的 repo 跟自己原创的 repo 混在一起？从管理项目的角度看，这种编排似乎很不方便，比如，明明很多 fork 过来的 repo 自己从来也没动过。所以，不好一眼区分哪些是自己真正贡献过的 repo（此处忽略 filter）。

带着这个疑问，查了下 star 和 fork 的区别。

star

一般做标记用，方便以后查看。同时，star 也能表达对项目的赞赏，所以从项目的 star 数量上大概能看出它的受欢迎程度。

fork

将项目所有内容包括项目的提交历史等信息拷贝到自己的账号下，当计划对项目做修改甚至有想法在未来往原仓库提交 PR 时会做 fork。

当弄明白这俩操作的区别之后，赶紧把以前 fork 过来的一些大的开源 repo 删掉了；-)

问题 2

最近 fork 了一个 scala-labs 的项目，里面有很多的练习题需要动手修改代码，本地改完代码后 commit/push 到 GitHub，但是在个人 profile 里只有一处 fork 的记录，并没有把 commit 统计到个人贡献图表里。

搜了一下，原来 GitHub 统计 commit 是有条件的，Why are my contributions not showing up on my profile?。

具体到我的例子，fork 来的项目，默认是不会计算 commit 贡献值的。只有当向源项目提 PR 并被 merge 之后才会体现为自己的贡献。另一个思路是与源项目完全解绑（detach the fork)，然后通过下面介绍的方式拷贝 项目。

如果不想提 PR，但是又想在项目上做改动，希望拷贝到自己账号下，又想将自己的 commit 体现到贡献值里，不能 fork，应该怎么做？

需要使用 mirror 的方式，参考 Duplicating a repository

假设需要将 wxl24life/scala-labs 拷贝到 wxl24life/scala-labs-exercise，新的项目已经在 GitHub 创建。

执行操作如下：

  1. $ git clone –bare https://github.com/wxl24life/scala-labs.git
  2. $ cd scala-labs.git
  3. $ git push –mirror https://github.com/wxl24life/scala-labs-exercise.git
