Java 关键字之 transient

在翻阅 Apache Flink 源代码的时候，单从代码观感上面看，有两大 Java 语言特性经常出现。一个是 Java 泛型，另一个就是大量使用 transient 关键字修饰类的成员变量。

泛型后面会专门写一篇文章进行复习。本篇文章简单介绍一下 transient 关键字的用法。

单词 transient 的含义

先看一下词典里这个单词的解释：

**transient**

/ˈtrænzɪənt;   [US]    ˈtrænʃnt; `trænʃənt/

adj lasting for only a short time; brief; fleeting 短暂的；片刻的；转瞬即逝的

从字面意思看，transient 有短暂片刻的意思。

看一下 官方 对这个关键字的解释：

Variables may be marked transient to indicate that they are not part of the persistent state of an object.

我们知道，当需要针对 Java 对象进行存储或者传输时，需要先对其做序列化，将对象转换成字节流。Java 默认对类里所有成员变量（非静态）都进行序列化。但是，实际应用中，我们可能只关心部分成员变量的值，例如只需要存储部分成员变量的值。

能不能只对部分成员变量进行序列化？

transient 关键字就是用来实现这个需求的。通过将类的成员变量使用 transient 修饰，我们显示声明了在对象序列化时，该成员变量不需要序列化。

通过 transient 修饰可以降低对象序列化和反序列化的开销，同时也能节省存储或传输时的带宽，尤其是当 transient 修饰的是一个较重的对象属性时，例如上面 Flink 代码里的几个对象成员。

另外，Java 里某个类如果需要序列化，需要满足两个条件：

  1. 要序列化的类本身实现了 java.io.Serializable 接口
  2. 类成员变量如果是对象属性，成员对应的类同样需要实现 java.io.Serializable 接口

前期我们在开始编写并部署 Flink 程序的时候，曾经出现过因为自定义的 Function 类成员没有实现序列化接口，而导致整个对象序列化失败的案例。当时查阅文档，采用的修复策略就是本文所介绍的 增加 transient 关键字修饰 。
